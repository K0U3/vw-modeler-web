# VW 自動モデリング Web ツール

意匠図DXFをアップロードすると、Vectorworks用の3Dモデリングスクリプトを自動生成する社内ツール。

## セットアップ（初回のみ・各自のMacで）

1. このリポジトリをダウンロード（Code > Download ZIP でも `git clone` でもOK）
2. ターミナルで:

```bash
cd vw-modeler-web/backend
python3 -m pip install -r requirements.txt
```

## 使い方

1. **`起動.command` をダブルクリック**（またはターミナルで `python3 -m uvicorn main:app --port 8000`）
2. ブラウザで <http://localhost:8000> を開く
3. 意匠図DXF（VWから「取り出す > DXF/DWG」で書き出したもの）をドラッグ＆ドロップ
4. 天井高を確認（空欄=図面のCH注記から自動検出）して「スクリプトを生成」
5. 生成されたスクリプトをコピーし、VWの **ツール > ツールマクロ > 新規(Python)** に貼り付けて実行

※ MUJI家具のシンボル自動配置を使う場合は、詳細設定の「家具ライブラリVWX」を自分のMacのMUJI家具VWXのパスに変更（無い場合は自動で簡易ボックス代替になります）

## チーム外への共有を防ぐ

`共有モードで起動.command` を使うとID/パスワード付きでLAN公開できます（ファイル冒頭のID/PASSWORDを書き換えて使用）。

## クラス（レイヤー）別の立ち上げルール

| クラス | 立ち上げ方 |
|---|---|
| 躯体 | 実頂点ポリラインをそのまま立ち上げ（二重外形は外周4辺帯に変換） |
| 壁・建具 | 壁＝閉ポリを立ち上げ。建具INSERT＝3方枠（両縦枠+上枠の簡易ボリューム）+ 扉パネル（図面の厚み20〜60mmをそのまま押し出し）+ 垂れ壁 |
| インナーサッシ | INSERT位置に共通窓ユニット（幅1500以上=掃き出し / 未満=腰高）。外周帯・内壁はサッシ位置で開口 |
| 断熱 | SOLID/閉ポリをそのまま立ち上げ |
| 天井・点線 | 梁ライン。平行2線はペアで1本に、単線は幅200に展開。**梁下端＝最寄りの「CH≒〇〇」注記**（無ければ天井からBEAM_D下がり） |
| 家具* | カタログ照合 → MUJIライブラリからシンボル自動インポート（該当なしは図面実寸の簡易ボリューム） |

## 窓の個別調整（生成後）

全窓に番号が振られる（`win(番号, 種別, ...)` とスクリプト末尾の窓一覧）。
生成スクリプト冒頭の `WIN_OVERRIDES` に番号をキーに `sill`（腰高）/`head`（まぐさ）等を書いて再実行すると、
前回の生成物を自動で消してから調整後のモデルを作り直す。

```python
WIN_OVERRIDES = {
    3: {'sill': 600, 'head': 2200},   # 窓3だけ腰高600・まぐさ2200に
}
```

## 玄関扉（共通・必ず1箇所）

玄関ラベル最寄りの開き戸ARC（半径550〜1200）を玄関扉と判定し、3方枠＋扉パネル（厚40）＋垂れ壁を立ち上げる。
ラベルが無い図面では最大半径のARCを採用。検出できない場合はダイアログに⚠表示。

## 家具認識の仕組み

1. 名前照合（MUJIシンボム名との部分一致・2-gram）
2. 寸法照合（ACIS実測寸法ベース・各辺平均±15%）
   - **部屋ラベルからのカテゴリ事前分布**: 洗面所→衛生 / キッチン→キッチン・家電 / 個室→ベッド等を優先
   - **ベッド標準寸法**（幅830/980/1200/1400/1580×長さ1950〜2080）は「ベッド」カテゴリ最優先
   - 一般レイヤーの家具フットプリントも走査（建物外形が取れた図面のみ）、入れ子輪郭は面積最大のみ採用
3. 該当なし → 図面実寸×FURN_BOX_Hの簡易ボリューム（無理にインポートしない）

**家具の固定ルール:**
- **位置整合**: 生成スクリプトは原点補正（OX/OY）を内蔵し、3Dモデルは元図面と同一座標に立ち上がる
- **向き**: 縦横入れ替えで寸法一致した家具は+90度回転して図面の向きに合わせる（INSERTの回転角も反映）
- **ベッド**: シンボルは使わず簡易ボリューム（高さ350）+ 枕（500×300×100・壁に近い端=頭側・幅1150以上は2個）
- **トイレ・キッチン・洗面台は1住戸に1つ**: 該当部屋のラベルに最も近い1件のみ配置し、重複は簡易ボリュームへ。部屋に合わない位置の水回りは寸法照合で割り当てない
- **段差マーク（Z形の線）・点線**は壁として立ち上げない

## モデリングのルール

- **天井高に既定値はない**: ユーザー指定 > 図面注記（"天井高2400"・"CH≒2400"・"CH:3670" 等を自動検出、最頻値を採用）> どちらも無ければ処理を止めて入力を求める（CLI は `--ch <mm>`）
- 部屋ラベル検出: トイレ/浴室/洗面所/キッチン/リビング/ダイニングの基本6室をチェックリスト表示（半角カナ・LDK表記対応）
- 躯体は実頂点形状で立ち上げ
- **内壁（間仕切り壁）**: 閉ポリライン壁（`壁・建具` レイヤー等）+ 薄壁(25〜60mm) + 平行LINEペア壁(壁厚50〜350mm) を立ち上げ。建具ARC位置でドア開口を抜き `DOOR_HEAD`〜CH の垂れ壁を生成
- RC躯体の二重外形（外形線+内形線）は外周4辺の壁帯に自動変換。塗り率の高い大型閉ポリ（家具外形等）は充填ブロック化を防ぐため除外してコメント報告
- 西面＝腰壁の連続窓 / 東面＝青線(建具 color=5)位置の FIX 窓 / 南北＝建具位置に腰窓
- **窓は全種共通の `win_unit`**: 枠＝FR見付×FD見込みの縦横角材ボリューム（上下枠・左右縦枠・方立・召し合わせ框）＋ ガラス＝GT厚の薄板ボリューム。引き違いは前後トラックにガラス2枚
- 窓種別ヘルパー: `ribbon_window`(腰壁連続) / `fix_window`(大FIX) / `hikichigai_window`(腰高引き違い) / `hakidashi_window`(掃き出し引き違い) — いずれも腰壁/垂れ壁 + `win_unit` の組み合わせ
- 梁: `梁` レイヤーの線/ポリラインを天井から `BEAM_D` 下げて立ち上げ
- **家具**: `家具*` レイヤー（前方一致）の INSERT/フットプリントをカタログ照合。無名グループ(グループ-N)もブロック実寸で寸法照合。生成スクリプトが **MUJI家具ライブラリVWXからシンボルを自動インポート**して配置し、取込不可なら W×D×H の簡易3Dボックスで代替（実測高さ入り）
- 外部階段・建物外は除外

## 家具カタログ（MUJI 等のシンボル集）

現行カタログ: 155件（h=高さ・z0=設置下端・vw_name=VWシンボル実名 付き。150件はACIS 3Dソリッドの実測寸法）。
旧版は `furniture_catalog.v1.backup.json` に保存。

再生成の手順（新しいライブラリを使う場合）:

```bash
cd ~/dev-projects/vw-modeler-web/backend
python3 build_catalog.py "<家具ライブラリ.dxf>"       # → 基本カタログ
# 高さ・実寸の付与は tools/extract_block_z.py → tools/enrich_catalog.py
# （venv + 最新ezdxfが必要。3DSOLIDのACISデータから実測する）
```

- VWシンボル名はDXFブロック名と原則同一（SJIS63バイト超の名前のみ末尾スペース差あり・検証済み）。**vw_name の末尾スペースは絶対に strip しない**
- 間取り図の家具は 名前照合 → 寸法照合（無名グループはブロック実寸で照合）の順で選定
- 生成スクリプトは `vs.BuildResourceListN(16, ライブラリVWX)` → `vs.ImportResToCurFileN` でシンボルを自動インポート → `vs.Symbol(名前, (x, y), 角度)` で配置（ポイントはタプル形式が公式仕様）
- インポート不可（ライブラリ不在・名前不一致）は W×D×H ボックス + z0 オフセットで代替
- 照合できない家具はスクリプト末尾にコメントで列挙（手動配置用）

## 図面が違うとき

画面の「図面が違うとき」を開き、窓の線色・建具レイヤー名・躯体レイヤー名を図面の規約に合わせる。
詳しい仕組みは `~/Documents/VW-3Dパイプライン/WORKFLOW.md` を参照。
